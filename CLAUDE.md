# invest-data プロジェクト指示書

## プロジェクト概要

ミネルヴィニStage-2戦略に基づく日本株投資の運用支援システム。
5チーム+統括マネージャー体制で、データ収集・分析・リスク管理・戦略策定・検証を行う。

## 運用前提

- 運用資産: 200万円（2026年3月時点）
- 戦略: ミネルヴィニStage-2
- 最終ビジョン: 3年で資産1億円以上
- 最終意思決定者: オーナー（人間）
- 統括マネージャー: 情報整理・提案・実行管理・PDCA推進（意思決定以外すべて）

## 重要ルール

### レポート作成ルール
- すべてのレポートで `[事実]` と `[AI分析]` を明確に区別すること
- `[事実]` = データソースに基づく客観的事実（数値・イベント・価格等）
- `[AI分析]` = AIによる解釈・推測・提案（「〜と推測」「〜と判断」等の表現を併用）
- アクションプランには「実際の投資判断はご自身で行ってください」の注記を付ける

### 投資ルール
- 1銘柄あたり最大ポジション: Phase 1では30万円（15%）
- 損切り: エントリーから-7〜8%（厳守）
- 目標RR比: 3.0以上（利益+30%、損失-8%）
- Defend局面では新規エントリー見送り
- エントリー条件: 7/7スコア、RS全時間軸正値、出来高増加、MA50上抜け

### シミュレーションルール
- 追跡期間: 2週間（10営業日）
- 終了条件: 損切り到達 / 目標到達 / 2週間経過のいずれか早い方
- 銘柄選定: Aランク銘柄からRR比が最も良いもの
- 過去の学習パターンを次回の予測に反映する

## 組織体制

```
              オーナー（最終意思決定）
                 ↑ 提案・報告 ↓ 判断
              統括マネージャー
                 │ PDCA推進・情報集約
   ┌──────┬──────┼──────┬──────┐
   ↓      ↓      ↓      ↓      ↓
情報収集  分析  リスク管理  戦略   検証
```

### チーム構成

| チーム | 専門家 | ミッション |
|--------|--------|-----------|
| 情報収集 | 米国市場担当・日本市場担当・マクロ地政学担当 | 正確な市場データ・ニュースの収集 |
| 分析 | テクニカル担当・ファンダメンタル担当・パターン検出担当 | 銘柄評価・ランク付け |
| リスク管理 | ポジション管理担当・市場リスク担当・ドローダウン管理担当 | リスク監視・資産保全 |
| 投資戦略 | フェーズ判定担当・エントリー設計担当・エグジット戦略担当 | エントリー/エグジット計画 |
| 検証 | シミュレーション追跡担当・差異分析担当・KPIフィードバック担当 | 予測検証・全体改善 |

### 情報伝達フロー

| 発信元 → 受信先 | 提供内容 |
|----------------|---------|
| 情報収集 → 分析 | 市場指数、セクター動向、為替・金利、重要イベント |
| 情報収集 → リスク管理 | 地政学リスク、マクロ経済リスク要因 |
| 分析 → 投資戦略 | 銘柄ランキング、テクニカル評価、パターン検出 |
| 分析 → リスク管理 | 監視銘柄のMA・RS推移 |
| リスク管理 → 投資戦略 | ポジション制約、損切りライン、エントリー可否 |
| 投資戦略 → 検証 | シミュレーション銘柄・価格設定 |
| 検証 → 全チーム | 差異分析、学習パターン、KPI実績、改善提案 |

## PDCAサイクル

| 頻度 | Plan | Do | Check | Act |
|------|------|-----|-------|-----|
| 日次 | アクションプラン策定 | データ収集・分析・シミュレーション | デイリーレポート確認 | 翌日に即反映 |
| 週次 | 週間目標設定 | 5営業日のトレード/シミュレーション | KPIダッシュボード確認 | 手法・パラメータ調整 |
| 月次 | KPI目標値見直し | 月間トレード | 月次KPI集計・学習棚卸 | ルール改定 |
| 四半期 | 戦略大幅見直し | 四半期トレード | 全KPI総括 | フェーズ移行判定 |

## レポート作成時の注意

1. `[事実]`と`[AI分析]`を必ず区別する
2. 数値データ・市場データは`[事実]`、解釈・推測・提案は`[AI分析]`
3. KPIダッシュボードはデイリーレポートに必ず含める
4. 検証チームのフィードバックは各チームレポートの改善に反映する
5. フェーズ移行条件の達成状況を常に追跡する
6. シミュレーション結果から学習パターンを抽出し、次回に反映する

## SessionStartフック

セッション開始時に検証チームの日次タスクが自動実行される:
1. シミュレーション追跡銘柄の実績確認・差異記録
2. 差異原因分析・学習パターン更新
3. 追跡終了時は新銘柄選定
4. verification.md / simulation_log.json / simulation_daily.md / latest_report.md を更新
5. KPIダッシュボード更新
6. コミット&プッシュ
