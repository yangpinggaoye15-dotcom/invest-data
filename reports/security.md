## 要対応事項

### 即時対応（Today）

1. **Claude Code バージョン確認（CVE-2026-21852）**
   - 開発チーム全員が使用する Claude Code が `v2.0.65` 以上であることを確認。未満の場合は即日アップデート。
   - `claude --version` で確認し、結果をセキュリティチームへ報告。

2. **GitHub Actions の Black Action 設定確認（CVE-2026-31900）**
   - `.github/workflows/` 内で `use_pyproject: true` を設定した Black Action が存在する場合は即時無効化または最新バージョンへ更新。
   - PR トリガーのワークフローでシークレットが参照されていないか合わせて確認。

### 今週中（〜2026-04-04）

3. **Gemini API キーのスコープ・公開状態の再確認**
   - `AIza…` 形式のキーが GitHub リポジトリ・ログ・設定ファイルに混入していないか `git log -S "AIza"` 等で全履歴スキャン。
   - Google Cloud Console にて API キーの利用制限（IP 制限・API 制限）が適切に設定されているか確認。

4. **Next.js / Vercel のバージョン確認と更新計画策定（CVE-2026-23864, 27978, 27980）**
   - 現行 Next.js バージョンを確認し、各 CVE の対応パッチが含まれるバージョンへの更新スケジュールを策定。

### 今月中（〜2026-04-30）

5. **認証情報の棚卸しとローテーション**
   - `kpi_log` 保持期間を3年に延長したことに鑑み、長期間ローテーションされていない API キー・トークン・パスワードを全件洗い出し。
   - `ANTHROPIC_API_KEY` および Gemini API キーのローテーションサイクルを明文化（推奨：90日以内）。

6. **GitHub Actions ワークフローの全件セキュリティレビュー**
   - `pull_request_target` トリガーの使用有無を確認（CVE-2026-1699 対策）。
   - シークレット参照箇所をすべて列挙し、最小権限原則に沿っているか検証。

## 推奨事項

### セキュリティ強化

| 優先度 | 推奨事項 |
|--------|---------|
| 高 | **Secret Scanning の有効化**: GitHub Advanced Security の Secret Scanning + Push Protection を有効化し、`AIza`, `sk-`, `ANTHROPIC_API_KEY` パターンをカスタムルールとして登録。 |
| 高 | **依存ライブラリの自動更新**: Dependabot または Renovate Bot を設定し、Python・npm パッケージの脆弱性パッチを自動 PR 化。`pyasn1 >= 0.6.3` を含む更新を優先。 |
| 中 | **OIDC 認証への移行検討**: GitHub Actions から Vercel・クラウドサービスへの認証を、長期有効な静的 API キーではなく OpenID Connect（OIDC）トークンベースに移行し、キー漏洩リスクを根本的に低減。 |
| 中 | **`kpi_log` データの保護強化**: 保持期間を3年に延長したことで蓄積データ量が増大。保存データの暗号化・アクセスログの取得・定期的なアクセス権限レビューを実施。 |
| 中 | **Next.js Server Action の CSRF 対策強化（CVE-2026-27978）**: `origin: null` を許可しないよう、カスタムミドルウェアで明示的にリジェクトする実装を追加。 |
| 低 | **ユーザー向け詐欺注意喚起**: サービス UI 上に「当社が SNS・LINE で投資勧誘することはありません」等の注意文を掲載し、ユーザー保護を強化。 |
| 低 | **耐量子計算機暗号（PQC）対応のロードマップ策定**: 金融庁の方針に沿い、2027年以降を見据えた PQC 移行計画の初期検討を開始。 |

### 運用プロセス改善

- **インシデントレスポンス手順の整備**: API キー漏洩を検知した際の対応フロー（即時無効化→ローテーション→影響範囲調査→報告）を文書化。
- **CSSA（サイバーセキュリティセルフアセスメント）対応**: 金融庁が 2026 事務年度より導入する CSSA の評価項目を事前に入手し、現状のギャップ分析を実施。
- **開発者セキュリティ教育**: 「バイブコーディング（AI 生成コード）」によるシークレット混入リスクについて、開発チーム全員へ周知徹底。
