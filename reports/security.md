## 要対応事項

### ❗ 緊急（今週中）

1. **LiteLLM使用有無の確認**
   - `requirements.txt` / `pyproject.toml` / `package.json` を確認し、`litellm==1.82.7` または `1.82.8` への依存がないか即時チェック
   - 該当する場合は安全なバージョンへのダウングレードまたはアップグレードを実施し、**ANTHROPIC_API_KEY / GEMINI_API キーをただちにローテーション**する

2. **GitHub ActionsでのTrivy使用有無の確認**
   - `.github/workflows/` 配下の全ワークフローファイルを確認し、`aquasecurity/trivy-action` または `aquasecurity/setup-trivy` の使用有無を調査
   - 使用していた場合、2026-03-19以降のCI実行ログを確認し、シークレット漏洩の痕跡がないかチェック。APIキーを即時ローテーション

3. **Pythonランタイムのパッチバージョン確認**
   - 使用中のPythonバージョンがCVE-2026-4519の修正パッチを含む最新版であることを確認

### ⚠️ 今月中

4. **GitHub Actions サードパーティActionのバージョン固定監査**
   - 全ワークフローにおいて、サードパーティActionがコミットSHA（例: `uses: actions/checkout@<SHA>`）で固定されているか確認。タグ参照（`@v3`等）はサプライチェーン攻撃に脆弱

5. **Next.jsバージョン確認（Vercel）**
   - CVE-2026-27980の対象バージョン（9.5.0〜15.5.13）を使用していないか確認。該当する場合はアップグレードを実施

## 推奨事項

### セキュリティ強化

```
優先度: 高
```
- **APIキーローテーションの定期化**: ANTHROPIC_API_KEY / GEMINI_API を四半期ごとにローテーションするポリシーを策定。今回の外部インシデントを受け、予防的に即時ローテーションを検討
- **依存関係の自動監査導入**: GitHub Dependabotまたは `pip-audit` / `safety` をCI/CDに組み込み、既知の脆弱性を含むパッケージを自動検出する
- **GitHub Actions OIDC認証の採用**: 長期的なシークレットの代わりに、OIDC（OpenID Connect）を使用してVercel・GCPへの一時的な認証トークンを取得する設計への移行を検討

### 運用・監視強化

```
優先度: 中
```
- **Vercel / GitHub アクセスログの監視強化**: MFA迂回リスクが高まっているため、異常なログイン・設定変更を検知するアラートを設定
- **AIコーディングエージェントの利用管理**: Claude Code等のAIコーディングツールを開発で使用している場合、CoWorker AIRDなどのリアルタイム監視ツールの導入を検討
- **インシデントレスポンス手順の整備**: APIキー漏洩発覚時の手順（ローテーション → 影響調査 → 報告）をドキュメント化し、チーム全員で共有

### ユーザー向け対策（本サービスが金融・投資情報を提供する場合）

```
優先度: 中
```
- SNS投資詐欺・ディープフェイクに関する注意喚起をサービスUI上に掲載することを検討
- 金融庁・IPAの注意喚起ページへのリンクを設置
